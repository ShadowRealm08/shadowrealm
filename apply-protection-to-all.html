Apply Source Code Protection to All Pages

This tool will help you apply source code protection to all HTML pages in your website. Follow the instructions below to implement the protection.

Step 1: Add Protection Scripts to Each HTML File

For each HTML file in your website, add the following code in the <head> section:

<!-- Source Protection Scripts -->
<script src="/JS/html-encryptor.js"></script>

And add the following code just before the closing </body> tag:

<!-- Source Protection Scripts -->
<script src="/JS/enhanced-protection.js"></script>
<script src="/JS/apply-protection.js"></script>

Step 2: Protect Sensitive Content

To protect specific sections of your HTML, add the data-protect attribute to the element:

<div data-protect>
    This content will be encrypted when the page loads.
</div>

You can also specify a custom encryption key:

<div data-protect data-protect-key="YourCustomKey">
    This content will be encrypted with a custom key.
</div>

Step 3: Pre-encrypt Content (Optional)

For maximum protection, you can pre-encrypt sensitive content:

<!-- 1. Create your content -->
<div id="sensitive-content" style="display: none;">
    <p>This is sensitive content that should be protected.</p>
</div>

<!-- 2. Add a script to encrypt it -->
<script>
    document.addEventListener('DOMContentLoaded', function() {
        const content = document.getElementById('sensitive-content').innerHTML;
        const encryptedContent = window.encryptHTML(content);
        
        // Create an element to hold the encrypted content
        const encryptedElement = document.createElement('div');
        encryptedElement.setAttribute('data-encrypted', encryptedContent);
        encryptedElement.setAttribute('data-key', 'ShadowRealm');
        
        // Replace the original element
        document.getElementById('sensitive-content').replaceWith(encryptedElement);
    });
</script>

Manual Implementation

If you prefer to manually implement the protection on each page, you can copy and paste the required code snippets into each HTML file.

Testing the Protection

After implementing the protection, test your website to ensure:

  • The website functions normally for regular users
  • Right-clicking is disabled
  • Keyboard shortcuts for viewing source are disabled
  • DevTools detection works properly
  • Encrypted content is properly decrypted when the page loads
